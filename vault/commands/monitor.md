# /monitor — Literature Monitoring & Automated Reviews

Continuous literature tracking with periodic synthesis. Tracks researchers, citation networks, and emerging topics.

## Commands

```bash
/monitor run                     # Full monitoring cycle
/monitor run --quick             # Quick check (high-priority only)
/monitor status                  # Show monitoring status
/monitor add-researcher [name]   # Track a new researcher
/monitor add-paper [id]          # Track citations to a paper
/monitor add-topic [topic]       # Track an emerging topic
/monitor review [topic]          # Generate literature review
/monitor gaps                    # Analyze knowledge gaps
```

---

## /monitor run — Full Monitoring Cycle

### Step 1: Load Configuration

Read `wiki/monitoring/config.md` to get:
- Tracked researchers (name, institution, alert level)
- Tracked papers (IDs for citation monitoring)
- Tracked topics (keywords, alert thresholds)

Read `wiki/monitoring/state.md` to get cursors (last check dates per item).

---

### Step 2: Check Tracked Researchers

For each tracked researcher:

```bash
python3 scripts/search_semantic_scholar.py --author "[Name]" --max-results 10
```

Compare results against the cursor in `state.md`:
- Filter to papers published after the last check date
- For each new paper, assess **thesis overlap** — how relevant is this paper to your research (from CLAUDE.md)?

Generate alerts:
- **high alert** researchers: Any new paper triggers an alert
- **medium alert**: Only papers with significant thesis overlap
- **low alert**: Collected for weekly summary

If thesis overlap is very high (>70%), escalate to **high** regardless of configured level.

Update the researcher cursor in `state.md`.

---

### Step 3: Check Citation Networks

For each tracked paper in config.md:

```bash
# Use Semantic Scholar for arXiv/DOI papers
python3 scripts/search_semantic_scholar.py --citations-of "[paper_id]" --days 30 --max-results 20

# Use PubMed for PMID papers (if pubmed enabled in config)
python3 scripts/search_pubmed.py --citations-of "[pmid]" --days 30 --max-results 20 --email "[from config]"
```

Compare against the cursor:
- Count new citations since last check
- For new citing papers, score their relevance to your research
- High-relevance citations get flagged

Update citation cursors in `state.md`.

---

### Step 4: Check Emerging Topics

For each tracked topic:

```bash
# arXiv (for CS/ML/physics topics)
python3 scripts/search_arxiv.py --keywords "[topic keywords]" --days 7 --max-results 30

# PubMed (for biomedical/life science topics — if pubmed enabled)
python3 scripts/search_pubmed.py --keywords "[topic keywords]" --days 7 --max-results 30 --email "[from config]"
```

Count papers matching the topic. If count exceeds the alert threshold, create a **topic surge** alert.

Also flag individual papers from major labs or with high early citation counts.

Update topic cursors in `state.md`.

---

### Step 5: Process Alerts

Collect all alerts from steps 2-4. Sort by alert level (high first), then by relevance score.

**High-priority alerts**:
1. Download the paper PDF
2. Run the full `/ingest` workflow (follow `commands/ingest.md`)
3. Mark as ingested in the alert

**Medium-priority alerts**:
- Add to the triage queue for the next daily briefing

---

---

### Step 7: Update State and Notify

1. Save updated cursors to `wiki/monitoring/state.md`
2. Append to `wiki/_log.md`
3. Output for OpenClaw:

If urgent alerts exist:
```
🚨 MONITORING ALERT

[N] urgent items:
• [Paper title 1] — [reason]
• [Paper title 2] — [reason]

📄 Report: wiki/monitoring/reports/YYYY-MM-DD.md
```

Otherwise:
```

📄 Report: wiki/monitoring/reports/YYYY-MM-DD.md
```

## /monitor add-researcher [name]

1. Search Semantic Scholar for the researcher to verify they exist
2. Add a new row to the `## Tracked Researchers` table in `wiki/monitoring/config.md`
3. Default alert level: **medium**
4. Confirm:
```
✅ Now tracking: [Name] at [Institution] (medium alert)
Change alert level in wiki/monitoring/config.md
```

## /monitor add-paper [id]

1. Look up the paper on Semantic Scholar to get title and current citation count
2. Add a new row to `## Tracked Papers` in `wiki/monitoring/config.md`
3. Initialize cursor in `wiki/monitoring/state.md`
4. Confirm:
```
✅ Tracking citations to: [Title] ([citation_count] current citations)
```

## /monitor add-topic [topic]

1. Add a new row to `## Tracked Topics` in `wiki/monitoring/config.md`
2. Ask what keywords to use (or infer from topic name)
3. Default alert threshold: 5 papers/week
4. Confirm:
```
✅ Tracking topic: [topic]
Keywords: [k1, k2, k3]
Alert threshold: 5 papers/week
```

## /monitor review [topic] — Literature Review Generation

### Step 1: Gather Sources

Search the wiki for all content related to `[topic]`:
- Summaries mentioning the topic
- Concept pages
- Entity pages
- Find related topics via wiki links

### Step 2: Analyze and Structure

Read each relevant source and identify:
- **Papers by year** (temporal distribution)
- **Papers by approach** (methodological clustering)
- **Key claims** with confidence levels
- **Consensus points** (claims appearing in 3+ papers)
- **Debates** (contradicting claims)
- **Gaps** (areas with thin coverage)
- **Trends** (temporal patterns)

### Step 3: Generate Review Document

Create `wiki/synthesis/reviews/[topic]-review.md`:

```markdown
# Literature Review: [Topic]

*Auto-generated: [date] | Sources: [N] papers | Last updated: [date]*

## Executive Summary
[3-5 sentences overview]

## 1. Introduction
[Topic] has emerged as a significant area in [field]. This review synthesizes [N] papers spanning [year range].

## 2. Historical Development
[Papers grouped by year with key developments per period]

## 3. Methodological Approaches
[Papers grouped by approach — what techniques are used, strengths, limitations]

## 4. Key Findings & Consensus
[Claims supported by multiple papers]

## 5. Open Debates
[Contradicting claims, different interpretations]

## 6. Research Gaps
[Areas with thin coverage, unanswered questions]

## 7. Trends & Future Directions
[What's gaining momentum, what's declining]

## 8. Relevance to My Research
[How this connects to thesis, potential applications, suggested next steps]

## References
[All papers cited, linked to wiki summaries]

---
*This review was auto-generated by `/monitor review`. Human review recommended.*
```

### Step 4: Update Wiki

- Add the review to `wiki/_index.md` under Synthesis > Literature Reviews
- Update concept pages with links to the review
- Log the action

## /monitor gaps — Knowledge Gap Analysis

Analyze the wiki for areas needing attention:

### Check 1: Thin Coverage
Concepts with fewer than 3 linked papers.

### Check 2: Stale Information
Papers older than 3 years with no recent confirmation.

### Check 3: Unanswered Questions
Open questions in `wiki/questions/` with no potential answers.

### Check 4: Missing Connections
Concepts that exist in isolation (no links to/from other concepts).

### Check 5: Thesis Gaps
Topics from CLAUDE.md thesis/keywords with less than 50% coverage.

Generate `wiki/monitoring/gap-analysis.md`:

```markdown
# Knowledge Gap Analysis — YYYY-MM-DD

## Critical Gaps (Thesis-Related)
[Topics from thesis with low coverage, with search suggestions]

## Thin Coverage
[Concepts needing more papers]

## Stale Information
[Old papers needing verification]

## Unanswered Questions
[Questions with no potential answers]

## Recommended Actions
1. This week: Address [N] critical gaps
2. This month: Review [N] stale papers
3. Ongoing: Monitor [N] thin concepts
```

Output:
```
🔍 Gap Analysis Complete

🔴 Thesis-critical gaps: [N]
🟡 Thin coverage: [N] concepts
📋 Stale info: [N] papers

Top priorities:
• [Gap 1 — suggestion]
• [Gap 2 — suggestion]

📖 Full report: wiki/monitoring/gap-analysis.md
```
